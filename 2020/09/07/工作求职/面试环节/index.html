面试环节

Posted on 2020-09-07   |   更新于 2020-10-11

自我介绍

我叫程瑞,家在安徽六安,本科是在本省安徽工业大学就读计算机,后来考研考到北航计算机读硕士,大实验室是新技术研究所,我们导师是做网格计算的,接触了一些分布式的理论和工具,研一至研二期间大约1年的时间里参与了实验室的一个web+数据挖掘的项目,自己在里面负责了中间层的功能,今年6月到9月在腾讯CSIG实习,负责路测数据元数据的管理

实验室项目

智慧科协项目,是中国科学技术协会打通协会内各部门的数据源,结合外部数据源,挖掘价值,我们实验室承接了该项目

5分钟

构成介绍：

目前分为两个系统 智慧科协大屏系统,有四个组成部分,党建 学术 科普 智库(这里涉及的数据源包括部门内本身的数据库,文件数据,以及外部科普馆的数据量等) 还有人才库系统(人才库需要解决的问题之一是消歧,中间参与了下),是从学术部分独立出来的,包含的人不仅仅只有学术人才(人才数量)

架构介绍

总体架构是利用vue构建前端,Django开发后端微服务,中间层利用kong网关做转发和限流,实现Oauth2标准的授权码模式支持第三方登录(智慧科协系统 人才库系统 科协一家app);
数据库层面:主要使用了mysql和mongodb存储数据,以及在网关和第三方登录系统 使用redis进行计数和满足插件功能,限制流量以及登录错误计数

20分钟

oauth2

Oauth1:从网站获取未授权的request_token,然后用户跳转登录授权request_token,后面根据request_token 获取access_token;

存在攻击者 在获取acces_token之后的回调地址被更改,然后获取access_token;由于access_token一般时间很长导致影响比较大

Oauth2:
流程：

  • 牛客网的登录页面有qq登录选项,点击之后跳转到qq登录页面跳转地址包括redirect_url/回调地址、response_type=code标识授权码模式,可以输入用户名和密码,qq后端验证完成之后会将code码附在回调地址上跳转;而每个使用qq第三方登录的app都必须先注册app_id和app_secret,加上前端传递过来的code码,可以从qq后端获取access_token,记住access_token可以获取用户的基本信息(具体信息内容可以在qq登录页面指定或者默认模式)
  • 登录过程中涉及的code token等信息分表存储

基于https(对http报文对称加密)更安全,code码不能直接获取信息,而且有时效(10s),只能使用一次,access_token有时效默认10分钟

kong网关

kong网关是基于Nginx的开源软件,可以通过接口和界面添加路由(最终通过热加载的方式生效),还可以添加插件实现想要的功能,我这里自定义了插件完成相关功能;最后根据kong网关的日志,基于ELK监控访问情况以及配置邮件预警.

插件功能/需求:

  • 实现单点登录/只允许一个用户使用,防止账号滥用,爬取数据(配合流量监控):
    • 对于携带token的请求,首先会验证token,首先尝试从redis中取,第一次请求时没有,之后查询token是否过期,如果过期返回登录,没有过期,则在redis里增加 < token 用户名 过期时间> 和 <用户名 token>键值对;需要注意的是可能已经存在了与该name对应的name token1键值对 那么意味着 如果 token token1不一致,则需要删除token1键值对 更新 name键值对
    • 对于携带token的请求验证token,利用check_token获取< token 用户名 过期时间> 存入redis< token 用户名 过期时间> 和 <用户名 token> 需要注意的是,可能已经存在name-token键值对了,意味着
    • 下次token过来,如果Redis中存在该 token 键,那么继续执行 否则 查询该token是否过期(2h),过期则返回登录(调用后端接口更新token)
  • 后端需要调用者信息,权限管理:
    • 直接传输用户信息不保证可信,所以需要机制确保用户名经过验证
    • 采取AES加密,利用token+时间key加密name信息,后端解密 aes_128_cbc_md5(中间遇到时区相差8h的问题,都转化为北京时间时间戳解决)

为什么不用redis存储token?

  • 默认方式是使用mysql持久化存储
  • 所存储的内容不仅是token和User的对应还涉及其他一些信息,存取比较复杂
  • 在kong处降低了访问的频度.

收获:

对于前后端的整体结构有了一定了解,负载均衡有了一定了解
对于微服务领域有一些了解

BUG:

  1. Oauth2登录完成之后跳转回访问的地址

跳转到 qq登录页面访问的接口时/oauth2/authorize 系统的默认实现类里面,会从url中提取中想要的keyvalue对,比如 redirect_url,grant_type,client_id 等等,其他都不会保存,完成之后直接重定向指redirect_url并附上code,经确认,该类这是默认实现,无法替换,后来想了想 改变了思路,取折中的方法,将redirect_url指向系统内部的path,然后后面附上原本的redirect_url 以及 跳转的来源refer字段,最后完成认证之后会转发到内部接口,再次转发即可,同时配合拦截记录refer字段

  1. 自动化脚本的坑
    表现是登录完成后,每隔1分钟 token失效,接口会报错,不稳定

首先开始mysql日志,发现每隔1分钟,mysql就会断开连接,然后重新连接:经检查spring 数据库的配置没有问题;
猜测是不是数据库连接池的问题,增大了一部分参数,没有任何效果
尝试删除jar包,重新生成运行新文件,发现没有问题
1分钟的时间间隔太过巧合,我想起来之前为了保证服务的稳健性,在crontab处设置 没1分钟执行一个shell脚本,shell 脚本的作用就是利用ps+grep 检查是否有进程 存在,如果不存在 就启动,由于脚本出错,导致误判不断启动服务,不断尝试连接数据库,并删除数据库里的token(应用启动时的脚本),导致波动

实习项目

我实习的部分是智慧出行部门,近期规划是提供自动驾驶的整个云平台工具链(采集,标注,管理,算法训练,metric评测等等)为此,我们需要不断的采集路测数据(侧重场景的数据输出),行测数据(侧重测试算法的输出);

由于 路测数据 规模大而且数据结构复杂,所以不便于直接使用而且有工具链上的许多环节需要人工去根据需求组合数据,所以我们需要在对数据处理之后,建立相关的元数据,配合自动化的流程操作以及人工的主观操作

5分钟

数据管理服务必要性

由于路测数据规模大,包含数十种传感器的输出数据,结构复杂而且总量达2T左右(30分钟的行程);所以不便于直接使用,而且实际使用中需要根据需求直观操作数据,所以需要抽象出元数据便于描述复杂数据(行程名车辆ID车辆名,起止时间,提交者,文件信息软件版本等等)
基于Spring Boot 开发后端服务,前端调用接口

自动流程描述

  • 车辆在行驶过程中,会记录数据,(出现异常点会通过kafka上报,调用管理服务的接口,创建异常数据);
  • 结束后手工通过数据管理服务的接口上传到COS中存储,同时创建行程元数据记录;
  • 上传完成后触发对于数据的初步解析任务(腾讯自研数仓TDW调度任务),解析的结果包括包括之前提到的(车辆信息,起止时间,文件信息大小 位置等),通过回调接口填充
  • 然后后端会触发分析行程数据的具体信息,识别出有价值的片段(5s) 特殊天气和路况的片段,据了解,方法时对于某一时刻的视频的识别,识别出是某一场景,会回调接口在管理服务的mysql中添加mark记录,包括片段的任务ID 起始时间 标记类型 地理信息 文件地址等等
  • 如果片段类型符合预设的要求,会触发后续的对该片段的评分机制,填充到该mark

手工操作

  • 复杂查询(涉及多条件 跨服务的查询):中间利用 redis 缓存了部分数据,避免重复加载
    • 是前后端数据粒度不对齐导致的; 后端在解析完行程后数据后,会间隔5s取一个时间点,存储对应的地理位置,图片等信息; 前端展示行程时分为多个小包展示,会将10分钟内的点作为一个小包内的数据,分页展示和请求
    • 当前端请求第二页的数据时,我不知道怎么向后端分页请求数据,因为前端的第二页对应的范围是什么?
    • 笨的方法是每次加载数据,或者利用redis,以查询条件为key,全部返回数据为value,然后下页请求,取就是,但是数据量还挺大的;最终选择的方法是第一次返回全部,记录全部数据每个小包的间隔点 存储在redis,下次以此为依据,按页请求即可。
  • 根据场景导出数据,算法工程师根据需求导出需要的数据 片段的联合以及形成数据的联合

BUG:

8月初的一天下午6点,快结束工作那天,发现接口出现错误,调用数仓的一个接口的返回数据fastjson反序列化为类时,出现错误包第一个字符错误

  • 我先查看日志,输出的String是正常的
  • 复制出来,json网站检验格式 正确
  • 我把字符串作为字符串常量,直接测试能否转换成类 发现可以
  • 有点懵逼,去看了 fastjson 源码,根据错误提示,找到错误的抛出位置,只能确定第一个字符不是{,还尝试更改fastjson的版本,发现没啥作用
  • 想着会不是因为反斜杠的问题,作为转义字符的反斜杠无法输出,在网上找了一个博客写的工具类,调用了StringEscapeUtils的方法把String 转化为Unicode字符,然后发现{前面出现\\

把情况与数仓对接的同时沟通之后,发现他那边下午把返回的结果序列化,我这边每次输入进来都会默认序列化,所以多次序列化,出现错误
